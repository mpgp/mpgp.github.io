Multiplayer Game Platform Введение

В современном мире высоких технологий все большую и большую популярность приобретают социальные сети, онлайн чаты и мессенджеры. Одним из повседневных действий для пользователя является общение в интернете. С развитием интернета и глобальным внедрением компьютерных технологий во все аспекты нашей жизни, интернет общение вытесняет живое общение. В конечном итоге это приводит к тому, что у каждого пользователя будет страничка в социальной сети или установленный мессенджер на устройстве. Все эти ресурсы можно вынести к одной функции – Web чат. Эти ресурсы предоставляют пользователю быстрый обмен информацией, удобный поиск собеседников по интересам, быстрое нахождение нужной и полезной информации. Всегда быть доступным для общения с друзьями. Актуальность темы обусловлена большим интересом пользователей к ресурсам данной тематики.


Цель и задачи

Целью команды является разработка Web ресурса, предоставляющего доступ к общению онлайн. Для достижения поставленной цели необходимо решить следующие задачи:

  • провести анализ предметной области, осуществить обзор существующих приложений в рассматриваемой предметной области;
  • разработать алгоритм работы приложения;
  • проработать структуру проекта, разбить проект на модули, каждый из которых будет выполнять одну из заявленных функций
  • выбрать инструментарий, определить среду, в которой будет создаваться приложение и язык, на котором будет написан код приложения;
  • написать основной код приложения и коды всех сопутствующих функций;
  • разработать удобный для пользователя интерфейс приложений;
  • разработать руководство пользователя, руководство программиста, обозначить системные требования;
  • протестировать полученное приложение и убедиться в ее корректности.

Выбор языка программирования

C# — представитель семьи языков программирования Microsoft. Он был создан в 2000 году как часть платформы Microsoft .Net framework. В использовании C# очень близок к Java, хотя говорили, что C# сочетает надёжность C++ с продвинутыми характеристиками Java.

На языке C# можно создать практически все типы приложений для большинства платформ.

Позиционируется как «лёгкий» редактор кода для кроссплатформенной разработки веб- и облачных приложений. Включает в себя отладчик, инструменты для работы с Git, подсветку синтаксиса, IntelliSense и средства для рефакторинга. Имеет широкие возможности для кастомизации: пользовательские темы, сочетания клавиш и файлы конфигурации. Распространяется бесплатно, разрабатывается как программное обеспечение с открытым исходным кодом

РАЗРАБОТКА ПРИЛОЖЕНИЯ Разработка диаграммы прецедентов

Диаграмма прецедента поможет описать функциональное назначение система или то, что система должна делать. Проектируемая система представлена в виде множества сущностей или актеров, взаимодействующих с системой с помощью прецедентов. При этом (actor) или действующим лицом называется любая сущность, взаимодействующая с системой извне. Это может быть человек, техническое устройство, программа или любая другая система, которая может служить источником воздействия на систему так, как определит разработчик, прецедент служит для описания сервисов, которые система предоставляет актеру.

Тестирование программы

Следующим пунктом выполнения проекта стало тестирование работоспособности готового приложения «Web-чат». Существует несколько стратегий тестирования приложения, в том числе черного ящика.

В данной стратегии приложение рассматривается как чёрный ящик. Целью тестирования ставится выяснение обстоятельств, в которых поведение приложения не соответствует спецификации. Для обнаружения всех ошибок в приложении необходимо выполнить исчерпывающее тестирование, то есть тестирование на всевозможных наборах данных. Для большинства приложений такое невозможно, поэтому применяют разумное тестирование, при котором тестирование программы ограничивается небольшим подмножеством всевозможных наборов данных. При этом необходимо выбирать наиболее подходящие подмножества, подмножества с наивысшей вероятностью обнаружения ошибок. В качестве исполняемой ветки программы было решено протестировать регистрацию пользователя.
